# Deriving signals natively from git history

**Status:** Design / planning
**Goal:** Make builder-profile produce a full profile for *any* Claude Code user, not just
the author. Today the headline metrics are read from pre-baked retro JSON snapshots that only
exist on the author's machine. This document describes wiring the already-present
`git_collector` / `correlator` modules into the pipeline so those metrics are derived directly
from `git log`, demoting the retro JSONs to optional enrichment.

---

## Problem

The current pipeline (`cli.py`) reads almost every headline signal from
`~/Code/**/.context/retros/*.json` via `retro_collector.aggregate_retros`. Those JSONs are
generated by a separate retro tool that only the author runs. For anyone else:

- `collect_retros` returns `[]`
- `aggregate_retros` returns an empty `BehavioralSignals`
- every git/volume/quality/temporal metric stays at zero/None

What still works for any user comes only from `behavioral.py` parsing raw transcripts:
prompt length, correction rate, politeness, question ratio, plan-mode %, top phrases,
model distribution, longest session, max parallel agents.

So a stranger gets the "steering style" cards but a blank slate for volume, quality, and
timing вЂ” and the archetype picker (which leans on the missing git metrics) misfires.

## Why this is mostly already solved

The plumbing exists and is unused:

- `discovery._resolve_real_path` resolves each project's real repo path from the transcript
  `cwd` and stores it on `ProjectManifest.real_path` (it even calls `_get_git_remote`).
- `parser` fills `session.cwd` and `session.branch` from the JSONL.
- `git_collector.collect_git_history(repo_path, since_epoch)` is complete and robust:
  author-email detection (config + most-frequent-committer fallback), `Commit` objects with
  `is_mine`, ISO dates, subjects, and per-commit `--numstat` в†’ `FileChange(path, added, deleted)`.
- `correlator` matches commits to sessions by time window (with a latent bug, see below).

Both `git_collector.py` and `correlator.py` have complete code and **zero callers**.

## Signal-by-signal coverage

| Retro signal | Native source | Effort |
|---|---|---|
| `total_commits` | `len([c for c in commits if c.is_mine])` | free |
| `total_insertions` | `sum(fc.added вЂ¦)` | free |
| `best_shipping_day` | `commit.date.weekday()` tally | trivial |
| `peak_hour` / `hourly_distribution` / `late_night_pct` | commit timestamps **or** `stats-cache.json` `hourCounts` (already computed by `stats_collector`) | trivial |
| `streak_days_max` | sorted commit dates, run-length | ~10 lines |
| `feat_pct` / `fix_pct` / `features_shipped` | parse `commit.subject` prefix (`feat:` / `fix:`) | ~10 lines, not in git_collector today |
| `ai_assisted_commits` | detect `Co-Authored-By: Claude` trailer (needs `%b` body in log format) | small format change |
| `hotspots` | frequency of `fc.path` across numstat | ~10 lines |
| `test_ratio_avg` | added lines in test-path files Г· total added | ~15 lines, path heuristic |
| `total_prs` | **not in `git log`** вЂ” parse merge commits (`Merge pull request #N`) or `gh` | medium |
| `coverage_pct` | **genuinely external** вЂ” real CI coverage, cannot come from git | n/a вЂ” stays in retro enrichment |

~70% of the headline signals are free or ~10 lines each. `total_prs` needs one extra source.
`coverage_pct` is the only signal that legitimately requires an external feed and is the reason
retros remain as optional enrichment rather than being removed.

## The change

### 1. New module `aggregate_commits`

A `aggregate_commits(commits: list[Commit]) -> BehavioralSignals` function holding the
derivation logic from the table above. This is the only net-new code (~80 lines). `git_collector`
returns `Commit` objects but does not itself compute feat/fix/streak/hotspots/test_ratio вЂ” that
logic lives in the external retro generator today and moves in-tool here.

### 2. Wire `git_collector` into `cli.py`

Between the existing discovery loop and synthesis:

```python
from builder_profile.git_collector import collect_git_history
all_commits = []
for manifest in manifests:
    all_commits.extend(collect_git_history(manifest.real_path, since_epoch))
git_sig = aggregate_commits(all_commits)   # then merge into `sig`
```

### 3. Merge policy

Retro JSONs win where present (they carry `coverage_pct`); `git_sig` fills every gap. This
makes retros pure enrichment instead of a hard dependency вЂ” exactly what unblocks other users.
When no retros exist, the profile is fully populated from git alone.

### 4. `total_prs` derivation

Parse merge commits (`Merge pull request #\d+`) from `git log`, or shell out to `gh pr list`
when available. Fold into `aggregate_commits`.

### 5. Fix `correlator._match_session` dead branch filter

The filter expression

```python
and (session.branch and c.sha or not session.branch)
```

is always truthy (`c.sha` is a non-empty string when a commit exists; the else arm is
`not session.branch`), and `Commit` has no branch field anyway вЂ” so branch-scoped correlation
is effectively unimplemented. The time-window match works, but if `correlator` is wired in for
per-session attribution (which would power native `loc_per_session_hour` and work-stream
narratives), this must be fixed or commits leak across branches that overlap in time.

`correlator` is **not** needed for the aggregate signals above; it is a separate, optional
enhancement for per-session attribution.

## Before / after data flow

### Before вЂ” retro JSONs are a hard dependency

```
~/.claude/projects/*.jsonl в”Ђв”Ђв–є parser + behavioral в”Ђв”Ђв–є steering signals only
~/Code/**/.context/retros/*.json в”Ђв”Ђв–є aggregate_retros в”Ђв”Ђв–є ALL headline metrics (Adam-only)
git_collector.py / correlator.py в”Ђв”Ђв–є orphaned, 0 callers
```

No retros = headline metrics all zero.

### After вЂ” git is the native backbone

```
~/.claude/projects/*.jsonl в”Ђв”Ђв–є discovery.real_path в”Ђв”Ђв–є git_collector.collect_git_history
                                                  в””в”Ђв”Ђв–є aggregate_commits в”Ђв”Ђв–є headline metrics
~/Code/**/.context/retros/*.json в”Ђв”Ђв–є aggregate_retros в”Ђв”Ђв–є MERGE (retro wins for coverage_pct)
```

Retros optional; profile fully populated for any Claude Code user.

## Scope summary

| | Before | After |
|---|---|---|
| headline metrics | retro JSONs (author-only) | git log (universal) |
| retro JSONs | hard dependency | optional enrichment |
| `git_collector` | orphaned, 0 callers | 3-line loop in `cli.py` |
| `aggregate_commits` | does not exist | new ~80 lines |
| stranger's profile | mostly empty | fully populated |
| still external | вЂ” | `coverage_pct` only |
